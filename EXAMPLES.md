## 1. Шифр Віженера

### Принцип роботи

Шифр Віженера - це поліалфавітний шифр підстановки. Кожна буква тексту зсувається на різну кількість позицій в алфавіті залежно від відповідної букви ключа.

**Ключ:** `CRYPTOGRAPHY`

### Приклад шифрування

**Оригінальний текст:**
```
The artist is the creator of beautiful things.
```

**Процес шифрування:**
```
Текст:  T h e   a r t i s t   i s   t h e   c r e a t o r   o f   b e a u t i f u l   t h i n g s
Ключ:   C R Y P T O G R A P H Y C R Y P T O G R A P H Y C R Y P T O G R A P H Y C R Y P T O G R A P
Шифр:   V y c   p k h o j t   x z   r j v   a g x o z f r   d m   z g r s i b t a c   t w p l i j
```

**Зашифрований текст:**
```
Vyc pkhojt xz rjv agxozfr dm zgrsibtac twplij.
```

### Рівень 2: Криптоаналіз

#### Метод Касіскі

**Мета:** Знайти довжину ключа шляхом пошуку повторюваних послідовностей

**Алгоритм:**
1. Знаходимо повторювані фрагменти тексту (наприклад, "abc" зустрічається двічі)
2. Вимірюємо відстані між повтореннями
3. Обчислюємо НСД (найбільший спільний дільник) цих відстаней
4. НСД дає ймовірну довжину ключа

**Результат для нашого тексту:**
- Передбачувана довжина ключа: 1 (може бути неточним через малий розмір вибірки)
- Реальна довжина: 12

#### Тест Фрідмана

**Мета:** Визначити довжину ключа через індекс збігу (IC)

**Формула індексу збігу:**
```
IC = Σ(fi × (fi - 1)) / (n × (n - 1))
```
де fi - частота кожної букви, n - загальна кількість букв

**Для англійської мови:**
- IC ≈ 0.065 (природний текст)
- IC ≈ 0.038 (випадковий текст)

**Формула довжини ключа:**
```
L = (0.027 × n) / ((n - 1) × IC - 0.038 × n + 0.065)
```

**Результат для нашого тексту:**
- Передбачувана довжина ключа: 9
- Реальна довжина: 12
- Похибка пояснюється розміром тексту

## 2. Шифр перестановки

### Рівень 1: Проста перестановка

**Ключ:** `SECRET`

**Принцип:**
1. Ключ сортується в алфавітному порядку
2. Визначається порядок перестановки

```
Ключ:     S E C R E T
Позиції:  0 1 2 3 4 5
Сортовано:C E E R S T
          2 1 4 3 0 5
```

**Порядок перестановки:** [2, 1, 4, 3, 0, 5]

### Приклад шифрування

**Оригінал:**
```
The artist is the creator
```

**Без пробілів:** `Theartististhecreator`

**Розбиття на блоки по 6:**
```
Theart
istist
hecreg  (доповнено до hcreatorXX)
```

**Перестановка символів у кожному блоці:**
```
Блок 1: T h e a r t → e h r a T t (позиції 2,1,4,3,0,5)
Блок 2: i s t i s t → t s s i i t
Блок 3: h e c r e a → c e e r h a
...
```

**Результат:**
```
rhTaetssiitteehrcafotorbiaetufiluhtnosgT...
```

### Рівень 2: Подвійна перестановка

**Ключі:** `SECRET` (перший), `CRYPTO` (другий)

**Процес:**
1. Застосовується перша перестановка з ключем SECRET
2. До результату застосовується друга перестановка з ключем CRYPTO

**Переваги:**
- Значно вища криптостійкість
- Складніше для криптоаналізу
- Довжина ключа фактично = LCM(len(key1), len(key2))

## 3. Табличний шифр

### Рівень 1: Табличний шифр

**Ключ:** `MATRIX`

**Принцип:**
1. Текст записується в таблицю по рядках
2. Стовпці нумеруються за алфавітним порядком букв ключа
3. Текст зчитується за стовпцями в цьому порядку

**Приклад:**

```
Ключ:    M A T R I X
Порядок: 4 0 5 3 2 1

Таблиця:
  M A T R I X
  4 0 5 3 2 1
  ---------
  T h e a r t  → зчитуємо: h,r,r,a,t,e (за порядком стовпців)
  i s t i s t
  h e c r e a
```

### Рівень 2: Комбінований шифр

**Метод:** Віженера → Табличний

**Ключі:**
- Віженера: `CRYPTOGRAPHY`
- Табличний: `CRYPTO`

**Процес:**
```
Оригінал → [Віженера] → Проміжний → [Табличний] → Фінальний шифр
```

**Переваги комбінованого шифру:**
1. **Підстановка + Перестановка** - класична надійна комбінація
2. **Складність аналізу** - частотний аналіз не працює
3. **Подвійний захист** - треба зламати обидва шифри

**Приклад першого рядка:**

```
Оригінал:  "The artist is the creator of beautiful things."
           ↓
Віженера:  "Vyc pkhojt xz rjv agxozfr dm zgrsibtac twplij."
           ↓
Табличний: "Vojzgaispcrwpfghrcxdpgpaedpahg,c,hiclknq..."
```

## 4. Порівняння шифрів

### Таблиця порівняння

| Характеристика | Віженера | Перестановка | Табличний | Комбінований |
|---------------|----------|--------------|-----------|--------------|
| **Тип** | Підстановка | Транспозиція | Транспозиція | Комбінований |
| **Складність ключа** | Довжина слова | Довжина слова | Довжина слова | 2 × довжина слова |
| **Зберігання пробілів** | ✓ | ✗ | ✗ | ✗ |
| **Зберігання регістру** | ✓ | ✓ | ✓ | ✓ |
| **Частотний аналіз** | Стійкий | Уразливий | Уразливий | Стійкий |
| **Криптостійкість** | Середня | Низька | Низька | Висока |

### Стійкість до атак

#### Шифр Віженера
- **Частотний аналіз:** Стійкий (розподіл частот згладжується)
- **Метод Касіскі:** Уразливий (довгі тексти)
- **Тест Фрідмана:** Уразливий (статистичний аналіз)

#### Шифр перестановки
- **Частотний аналіз:** Уразливий (частоти не змінюються)
- **Аналіз біграм:** Уразливий
- **Перебір ключів:** Уразливий (факторіал довжини)

#### Табличний шифр
- **Частотний аналіз:** Уразливий
- **Аналіз структури:** Середньо уразливий
- **Перебір порядків:** Уразливий

#### Комбінований шифр
- **Частотний аналіз:** Стійкий
- **Складність:** Потрібно зламати обидва шифри
- **Час криптоаналізу:** Значно збільшений

## 5. Висновки

### Основні висновки

1. **Поліалфавітні шифри (Віженера)** ефективніше протистоять частотному аналізу, ніж моноалфавітні

2. **Шифри перестановки** зберігають частотні характеристики тексту, але змінюють порядок символів

3. **Комбінація різних типів шифрів** (підстановка + перестановка) дає найкращий результат

4. **Довжина ключа** критично важлива - чим довший ключ, тим складніше криптоаналіз

5. **Криптоаналіз** класичних шифрів можливий завдяки статистичним методам (Касіскі, Фрідман)

### Практичне застосування

**Історичне значення:**
- Шифр Віженера використовувався століттями
- Перестановочні шифри застосовувались в армії
- Комбіновані шифри - основа сучасної криптографії

**Сучасний контекст:**
- Принципи підстановки → блокові шифри (AES)
- Принципи перестановки → P-блоки в DES
- Комбінування → раунди в сучасних шифрах
